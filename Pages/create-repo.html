Creating a Git Repository

Git repo is a storage area where your project files and their complete version history are tarcked and managed by git

By default Git does not track every files present in your local system. We need to tell git which files we want to track by initializing a .git folder inside our Project folder.

Initializing a Repository

To create a new repository in an existing project directory, use the git init command:

git init

This command sets up a Git repository in your current directory. A hidden .git directory will be created to store all the version history and configuration files.

You can see .git folder by :

$ls -lah
 total 88k
 drwxr-xr-x 1 Asus 197121 0 Sep 9 21:32  ./
 drwxr-xr-x 1 Asus 197121 0 Sep 9 21:22  ../
 drwxr-xr-x 1 Asus 197121 0 Sep 9 21:32  .git

Cloning an Existing Repository

If you want to work on an existing project, you can clone it from a remote source (e.g., GitHub):

git clone https://github.com/Ansh-Vikalp/GitMastery.git

You can either clone your remote repo from your account to your local system or clone someone else remote repo by first forking it to your account then using the HTTPS/SSH code to clone it to your local system.
