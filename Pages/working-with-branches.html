Working with Branches

What are Branches?

Branches in Git are separate workspaces that allow you to work on specific tasks (such as feature development or bug fixes) without affecting the main codebase. By default, the main branch is usually named main or master, and developers often create additional branches to keep the development isolated. This allows for parallel development and easy integration later through merging.

Creating a New Branch

To create a new branch, use the git branch command followed by the branch name:

git branch <branch-name>

This will create a new branch but will not switch to it automatically. If you want to create and switch to the branch immediately, use:

git checkout -b <branch-name>

Switching Between Branches

Once you have multiple branches, you can switch between them using the git checkout command:

git checkout <branch-name>

Listing Branches

To see a list of all the branches in your repository, use the git branch command:

git branch
* master
testing

The current branch you're on will be indicated by an asterisk (*) next to its name.

To see more detail verbose of branch is to check which branch is set as upstream to orgin:

git branch -vv
* master e391cc5 [origin/master] added codes
testing 1e20859  One file renamed

Deleting a Branch

Once a branch is no longer needed, you can delete it to keep your repository clean. To delete a branch that has already been merged, use:

git branch -d <branch-name>

If the branch has not been merged and you want to force delete it, use:

git branch -D <branch-name>

Renaming a Branch

If you want to rename the current branch, you can do so with:

git branch -m <new-branch-name>

Remote Branches

Branches that exist on a remote repository (such as GitHub) can be fetched with git fetch. To switch to a remote branch, use:

git checkout -t origin/<branch-name>

This will create a local tracking branch that follows the remote branch.
