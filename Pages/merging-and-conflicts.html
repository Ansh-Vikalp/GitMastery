Merging & Resolving Conflicts

What is Merging?

Merging in Git combines changes from one branch into another. This is typically done when a feature branch is ready to be integrated into the main branch. Merging preserves the history of both branches and helps in collaboration by allowing parallel development.

How to Merge Branches

To merge changes from another branch into your current branch, use the git merge command:

git merge

This merges the specified branch into the branch you are currently on. For example, to merge a branch called feature-branch into the main branch: 

git checkout main
git merge feature-branch

The merging process combines the histories and changes of both branches.

Fast-Forward Merge

A fast-forward merge happens when the current branch has no new commits since the branch you're merging was created. Instead of creating a merge commit, Git will simply move the branch pointer forward, effectively ‚Äúfast-forwarding‚Äù the branch:

git merge feature-branch

In this case, the commit history is linear, as the branch pointer moves to the latest commit of the feature branch.

3-Way Merge

A 3-way merge occurs when both branches have made unique changes since they diverged. Git uses the common ancestor of the two branches to merge changes, but a new commit (merge commit) will be created to record the combination of changes from both branches:

git merge feature-branch

This type of merge happens when there are divergent changes, and Git needs to reconcile differences between the branches.

Merge Conflicts

Merge conflicts occur when both branches have made changes to the same lines of code or file. In such cases, Git will pause the merge and allow you to resolve the conflict manually. When a conflict happens, Git will insert conflict markers into the affected files, and you will need to decide which changes to keep.

<<<<<<< HEAD
// Your changes
=======
 // Other branch changes
>>>>>>> feature-branch

To resolve the conflict, you must edit the file and choose which lines to keep or combine, then remove the conflict markers.

How to Resolve Merge Conflicts

After resolving the conflict in the file, you need to stage the changes and commit them. Here‚Äôs the process:

  1. Edit the conflicting files to resolve the conflicts.
  2. Stage the resolved files using git add:
    3. git add
  3. Commit the merge using git commit:
    4. git commit

Once committed, the merge is complete, and the combined changes will be part of the current branch.

Viewing Merge Conflicts

To see which files have conflicts during a merge, use git status. It will show you which files are unmerged and need attention:

git status

Aborting a Merge

If the merge is too complex or you decide not to merge at all, you can cancel the merge process with:

git merge --abort

This will revert your working directory to the state before the merge attempt.

Commit Messages During Merge

When performing a 3-way merge, Git will generate a default commit message for the merge commit, which you can modify if necessary. This message usually indicates which branch was merged into the current branch.
