What is Branching in Git?

Branching allows you to create separate lines of development within a project. You can work on new features, bug fixes, or any changes independently without affecting the main project code.

Important Commands for Branching

  • 1. Create a Branch: git branch branch-name creates a new branch.
  • 2. Switch to a Branch: git checkout branch-name allows switching to an existing branch.
  • 3. Create and Switch: Use git checkout -b branch-name to create and switch to a new branch in one step.
  • 4. List Branches: git branch shows all local branches.

What is Merging in Git?

Merging allows you to combine the changes made in one branch into another, usually integrating a feature branch into the main branch.

Important Commands for Merging

  • 1. Merge a Branch: git merge branch-name merges the specified branch into the current branch.
  • 2. Abort a Merge: If a conflict arises during a merge, you can abort it using git merge --abort.
  • 3. Resolve Merge Conflicts: If there are conflicts, Git will mark them in the files, and you can resolve them manually, followed by git add and git commit.

Pushing and Pulling Branches

  • 1. Push a Branch: Use git push origin branch-name to push a local branch to the remote repository.
  • 2. Pull a Branch: git pull origin branch-name fetches changes from a remote branch and merges them into the current branch.
