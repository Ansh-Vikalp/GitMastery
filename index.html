What is Git?

Git is a free and open-source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. It allows developers to track changes in source code and collaborate with others on shared projects.

How to Use Git

  • 1. Initialize a Repository: Run git init in your project folder to create a new Git repository.
  • 2. Track Changes: Use git add file-name to stage changes for commit.
  • 3. Commit Changes: Save changes to the repository with git commit -m "commit message".
  • 4. View History: Use git log to see the commit history and details of each commit.
  • 5. Branching: Create a new branch using git branch branch-name and switch to it using git checkout branch-name.
  • 6. Merge Branches: Combine changes from different branches with git merge branch-name.
  • 7. Push to Remote: If you're working with a remote repository (like GitHub), use git push origin branch-name to upload your changes.

Repeted Modification

  • 1. Initial Modification: You modify the file and stage it (git add file). At this point, the staging area holds the version of the file with the first modification.
  • 2. Further Modification: You modify the file again and stage it once more. This overwrites the previous version of the file in the staging area with the latest changes.
  • 3. Commit Changes: When you commit the changes git commit -m "This is new", Git will commit the version of the file that is currently in the staging area, which is the file with the most recent modification.
  • 4. View History: Use git log to see the commit history and details of each commit.

Working with Remote Repository

We can simple use git pull
  • 2. git push: The git push command is used to send your committed changes to the remote repository git push will pushes the local changes in the master branch to the master branch of the remote repository.
  • 3. git clone: The git clone command is used to create a copy of an existing remote repository to your local machine git clone repository-url The cloned repository contains all files along with tracked history of all commits so a tester can use these commits by git show commit-id:index.html or cat index.html to see content in latest commit.
