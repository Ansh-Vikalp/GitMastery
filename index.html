Introduction to Git & Version Control

What is Git?

Git is a distributed version control system (VCS) designed to handle everything from small to large projects with speed and efficiency. It tracks changes in source code and allows multiple developers to collaborate on the same project.

It was created by Linus Torvalds in 2005 to manage development of Linux Kernel. Although, Git is highly popular for managing code, but can also be used to track changes in any files types- text doc., images etc. Git is a tool that helps multiple people work on the same project code by tracking and managing changes to files.

Why Version Control Matters?

Version control is essential for tracking changes, collaboration, and maintaining a history of code changes over time. It allows developers to work together without overwriting each other's work.
Evertime anyone is making changes whether it is adding a sentence to document or alternating a line or even deleting some lines then the VCS records and saves the outcome with accurate timestamp.

Centralized vs Distributed VCS

  • Centralized VCS: One central server stores all files, and developers commit changes directly to it. Examples: SVN, CVS.
  • Distributed VCS: Every developer has a local copy of the project. They can work independently and sync changes. Example: Git.

Key Concepts

  • Repository: A directory that holds the project's files and the history of all changes.
  • Commit: A snapshot of your project's current state.
  • Branch: A version of the project used to work on new features without affecting the main codebase.
  • Merge: Combining different branches into one.

Key Features

  • Version Control:Git keeps a detailed history of all changes made to a project. This includes when the change was made, who made it, and why (through commit messages).
  • Distributed System:Each developer has a complete copy of the entire project and its history on their local machine. This means that Git does not rely on a central server, and changes can be made and tracked even offline.
  • Collaboration: Multiple developers can work on the same project, even on the same files, without overwriting each other's work. Git handles changes by providing tools like branches and merges.
  • Branching and Merging: Git allows developers to create branches to work on features or fixes in isolation from the main codebase. Once the changes are complete, the branch can be merged back into the main project.
  • Snapshots, Not Differences:Git saves entire snapshots of your file system rather than recording changes line by line. This makes it more efficient in tracking history and helps in reverting back to any previous version of the project.
  • Speed and Efficiency: Git is designed to handle projects of all sizes efficiently, from small personal projects to large enterprise-level repositories with millions of files and developers.
