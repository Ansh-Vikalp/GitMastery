Staging Files

Before creating a commit, you need to stage the files you want to include. Use the git add command to stage files:

git add <filename1> <filename2> <filename3>

To stage all files, use:

git add .

Committing Changes

Once your changes are staged, commit them to the repository with a descriptive message:

git commit -m "Your commit message"

Commit messages should be clear and concise, describing the changes made in the commit.

Viewing Commit History

To view the history of commits in the repository, use the git log command:

git log

This command will show a list of commits, including the commit ID, author, date, and commit message.
